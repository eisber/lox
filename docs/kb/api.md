## API – Programming Interface

The following information describes the programming interface (API) of the Miniserver:

[**Communicating with the Miniserver**](https://www.loxone.com/wp-content/uploads/datasheets/CommunicatingWithMiniserver.pdf)

– describes the basics of communication with the Miniserver.

[**Structure File**](https://www.loxone.com/wp-content/uploads/datasheets/StructureFile.pdf)

– describes the structure file.

[**Usermanagement**](https://www.loxone.com/wp-content/uploads/datasheets/UserManagement.pdf)

– describes [User and Rights Management](https://www.loxone.com/enen/kb/user-and-rights-management/).

[**Operating Mode Schedule**](https://www.loxone.com/wp-content/uploads/datasheets/OperatingModeSchedule.pdf)

– describes the management of [Operating Times](https://www.loxone.com/enen/kb/operating-times-periphery/), to activate [Operating Modes](https://www.loxone.com/enen/kb/operating-modes/).

[**PMS & Access API**](https://www.loxone.com/dede/wp-content/uploads/sites/2/2025/04/pms-access-api.zip)

– describes the integration of a Property Management System (PMS) into Loxone Technology.

This information is intended for experienced developers who develop third party software based on the Loxone Miniserver.

Please note that Loxone does not provide technical support for the API.

If necessary, this information is updated when Loxone Config or the Miniserver is updated.

We reserve the right to make changes to the interfaces during updates. We recommend that developers of third-party software always check the current API descriptions.
